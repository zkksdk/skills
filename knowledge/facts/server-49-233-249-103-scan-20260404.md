## 四、接入向导（setup.html）Redirect Bug 分析

### 现象
用户点击 AgentHub 前端 `/owner` 页面的「打开部署向导」按钮 → 浏览器打开新标签页 → URL 变成 `https://790h3zymqjwf.space.minimaxi.com/`（外部服务器）

### 技术分析

**代码来源** (Dashboard.tsx，第159行):
```tsx
💡 需要帮助？<a href="/setup.html" target="_blank" ...>打开部署向导</a>
```
编译后 JS 中的按钮实现：
```javascript
onClick: () => window.open(`/setup.html`, `_blank`)
```

**本地服务验证**（实测）：
```
curl http://127.0.0.1:8080/setup.html
→ HTTP/1.1 200 OK
→ Content-Type: text/html; charset=utf-8
→ （返回本地 setup.html 内容）
```

**dist 中无硬编码外部 URL**：
```bash
grep -r "790h3zymqjwf\|minimaxi\.com" /root/agenthub/web/agenthub-web/dist/
→ (无匹配)
```

**结论**：本地服务器配置正确，`/setup.html` 正常从本地提供。Redirect 可能在以下位置发生：
1. **浏览器缓存**：用户浏览器缓存了旧的 JS bundle
2. **CDN/网关层**：OpenClaw 部署环境（MaxClaw 云端托管）可能在网关层面有重定向规则
3. **网络层**：DNS 或代理设备拦截了请求

### 建议修复方案

**方案 A（推荐）**：在 setup.html 头部加入跳转保护脚本：
```html
<script>
if (location.hostname !== 'localhost' && location.hostname !== '127.0.0.1') {
  // 如果不是本地访问，检查是否应重定向
}
</script>
```

**方案 B**：检查 MaxClaw 云端托管的网关配置，看是否有针对 `/setup.html` 的重定向规则

**方案 C**：清除浏览器强缓存，让浏览器重新请求最新的 JS bundle
