# context-guardian — 上下文守护者 & 知识库

## 核心架构

```
/workspace/knowledge/
  index.md          # 知识索引（所有条目的入口）
  entities/         # 实体知识（人、项目、工具、概念）
    *.md
  decisions/        # 决策记录
    *.md
  facts/            # 用户事实与偏好
    *.md
  会话存档/          # 按会话归档的对话摘要
    YYYY-MM-DD/*.md
```

## 规则：全自动抽取（不等用户说）

**每次对话结束（每轮回复后自动判断）**，识别并保存：

- 用户提到的**事实**（"我的服务器是 CentOS" → facts/server.md）
- 用户做的**决策**（"用这个方案" → decisions/2026-04-02.md）
- 新出现的**实体**（项目名/工具名/地址 → entities/*.md）
- 任何值得**未来参考**的信息

## 知识抽取标准

**必须存：**
- 用户明确说的偏好、约束、要求（"只用免费方案"）
- 重要决策及原因（"选了方案A因为..."）
- 项目进度和状态
- 遇到的问题及解决方案
- 地址/链接/路径/ID等 identifiers
- 任何人名/用户名/项目名的第一次出现

**不存：**
- 闲聊、问候
- 重复确认（"好的"、"收到"）
- 工具调试过程（存结论不存过程）
- 临时测试内容

## 知识库读写接口

### 存知识 — "记一下..."

收到用户要求记忆信息时，立即存入对应分类文件，格式：

```markdown
# [实体/主题名]

## 基本信息
- 添加时间：YYYY-MM-DD
- 来源：当前对话

## 内容
（知识正文）

## 标签
#标签1 #标签2
```

### 查知识 — "知识库里有没有..."

1. 读取 `index.md` 扫描相关标签/标题
2. 用 `memory_search` 语义搜索 `/workspace/knowledge/` 下所有 .md
3. 返回匹配结果 + 出处文件路径

### 知识检索流程

当用户问"知识库里有..."时：

```bash
# 1. 语义搜索
memory_search(query="用户问题关键词", maxResults=5)

# 2. 补充文件扫描
grep -r "关键词" /workspace/knowledge/ --include="*.md" -l

# 3. 合成回答，注明出处
```

## 知识库索引（index.md）

每次存入新知识，同步更新 `index.md`：

```markdown
# 知识库索引

## 最近更新
- 2026-04-02 | entities/servers.md | 更新服务器列表
- 2026-04-02 | decisions/git-strategy.md | 确定用下载而非git push

## 实体
- servers: 139.224.44.184 (主), 49.233.249.103 (副)

## 项目
- AgentHub部署：进展中

## 待补充
- [用户技能/工具偏好]
```

## Running Summary（滚动摘要）

维护 `/workspace/context-guardian/running-summary.md`，每次会话后自动更新。

## 与 context-guardian 的配合

- **context-guardian** = 保证上下文不爆（上下文管理）
- **knowledge-base** = 保证知识不丢（知识沉淀）

两者同时运行：
- 每次回复后自动判断：存知识 OR 压缩上下文 OR 两者都做
- Running Summary 是上下文层，知识库是持久化层

## 触发词

**存知识：**
- "记一下..."
- "把这个存到知识库"
- "记住，我的..."

**查知识：**
- "知识库里有没有..."
- "查一下知识库"
- "之前有没有说过..."
- "我的...是什么"

**整理知识：**
- "整理一下知识库"
- "同步记忆"
- "更新知识库"

## 防爆优先级

1. **同时进行**：回复后自动存档（低成本）+ 检查 token（高成本）
2. **token > 80%**：停止长回复，强制先压缩
3. **token > 95%**：立即存档 + 摘要，释放窗口
